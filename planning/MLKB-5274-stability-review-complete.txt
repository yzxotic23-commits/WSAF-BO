3. KEEP / FIX / REMOVE LIST
================================================================================

KEEP (works or worth keeping on 1.0.21 baseline)
------------------------------------------------
- Desktop app with QR linking from sidebar
- Start and stop feeding from the main screen
- Chat preview and activity log during feeding
- Settings for language, delays, and proxy list
- Saved WhatsApp sessions between app restarts
- AI via Codex or local Ollama fallback
- Windows installer and GitHub release pipeline
- User data stored outside install folder (Windows + Mac)
- Rollback release v1.0.21-restored as current production baseline

FIX (planned — matches Jira tasks 2–6 next week)
--------------------------------------------------
- Phone number login stability and clear user messaging
- Proxy list load, save, test, and per-account assignment
- Start feeding reliability and sidebar "linked" accuracy
- Profile name sync after successful QR link
- Activity log noise and readability
- Mac update path: manual DMG download with clear in-app guidance
- Reconnect accounts automatically after feeding ends
- Error messages in plain English on the main screen

REMOVE (do not bring back without redesign)
-------------------------------------------
- Recommended use of versions 1.0.22 through 1.0.32
- Mac in-app auto-install via Squirrel on unsigned builds
- Post-build ad-hoc signing that broke update validation
- QR scan output inside activity log
- Pairing UI behaviour that blocks QR reconnect
- Update manifest publishing without file name verification
- Duplicate or conflicting proxy logic between desktop and background process

================================================================================
4. STABILITY DEFINITION NOTE
================================================================================

WSAF is "stable" when ALL of the following are true on v1.0.21-restored:

1. Both WhatsApp accounts can be linked by QR without getting stuck.
2. The main screen shows both accounts as linked with correct readiness to feed.
3. Start feeding runs at least one full pair session with visible messages.
4. Stop or natural end of feeding brings accounts back online without new QR scan.
5. Activity log stays human-readable during login and feeding.
6. Proxy list can be saved and tested from Settings (or direct connection works).
7. Windows users can install and run from the official GitHub release.
8. Mac users can install from DMG and open the app (with right-click Open if needed).
9. Mac users are not forced through broken in-app auto-update signature errors.
10. No recommended production use of versions 1.0.22–1.0.32 until a new QA-passed release.
