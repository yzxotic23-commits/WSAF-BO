SECTION C — QUICK COPY FOR JIRA COMMENT
================================================================================

Error history summary (WSAF):

User hit login failures (QR stuck, pairing invalid), Start feeding disabled,
unreadable activity log, and feeding flow confusion on builds v1.0.27–1.0.32.
Mac update failed with 404 missing file, then code signature validation errors
on v1.0.31–1.0.32. Team rolled back to v1.0.21-restored. Dev environment errors
(npm PATH, Electron, Codex) were fixed separately. Open items: phone login,
proxy load, feeding UX messaging, Mac manual update only until next stable
release. Full log attached: MLKB-5274-error-history-log.txt

================================================================================
SECTION D — RETROSPECTIVE SUMMARY (OPTIONAL JIRA FIELD)
================================================================================

What went wrong:
New features and fixes between v1.0.22 and v1.0.32 were released without
full regression on both Windows and Mac. Login and feeding changes conflicted
with each other. Mac auto-update was attempted on unsigned builds and failed
repeatedly.

What went right:
Rollback to v1.0.21-restored was completed. GitHub release pipeline still
produces Windows and Mac installers. User data and sessions were preserved
across reinstall.

Decision:
Use v1.0.21-restored as baseline. No v2.0 until stability checklist passes.
Mac updates via manual DMG until signing strategy is decided.
